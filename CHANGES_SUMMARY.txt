DATABASE CHANGES
================================================================================

TABLES CREATED:
===============
1. pipeline_stages
   - Configurable deal pipeline stages
   - Columns: id, name, pipeline_id, position, description, timestamps
   - Purpose: Better pipeline management

2. activities
   - Comprehensive activity tracking (calls, emails, meetings, notes)
   - Flexible entity linking (Company, Contact, Deal, Project)
   - Columns: activity_type, title, description, status, priority, entity_ids, timestamps
   - Purpose: Central communication log

3. entity_notes
   - Notes linked to any entity
   - Flexible entity linking (Company, Contact, Deal, Project)
   - Columns: title, description, entity_ids, priority, is_important, timestamps
   - Purpose: Entity-specific documentation

4. entity_files
   - File associations with any entity
   - Links existing files to Company/Deal/Contact/Project
   - Purpose: Document management across entities

5. project_tasks
   - Tasks within projects
   - Columns: title, description, project_id, status, priority, assigned_to, due_date, timestamps
   - Purpose: Project deliverable tracking

6. project_team
   - Team member assignments to projects
   - Columns: project_id, user_id, role, allocation_percentage, timestamps
   - Purpose: Resource allocation and team management

7. project_timesheets
   - Time tracking for projects
   - Columns: project_id, user_id, work_date, hours_worked, description, timestamps
   - Purpose: Billable hours & resource tracking

8. estimation_line_items
   - Line items within estimations
   - Columns: estimation_id, item_name, quantity, rate, discount%, tax%, amounts, timestamps
   - Purpose: Detailed quotation items with automatic calculations

9. contact_tasks
   - Tasks specifically for contacts
   - Columns: title, description, contact_id, status, priority, assigned_to, due_date, timestamps
   - Purpose: Contact-specific task management

TABLES ENHANCED:
================
1. deals
   + Added: pipeline_stage_id (Foreign Key → pipeline_stages)
   - Allows explicit stage tracking separate from string stage field
   - Better pipeline management and filtering

2. estimations
   + Added: deal_id (Foreign Key → deals)
   + Added: subtotal (DECIMAL 15,2)
   + Added: discount_percentage (DECIMAL 5,2)
   + Added: tax_percentage (DECIMAL 5,2)
   + Added: total (DECIMAL 15,2)
   - Enables better financial tracking
   - Automatic calculation support
   - Clear deal → estimation → invoice flow

3. invoices
   + Added: estimate_id (Foreign Key → estimations)
   - Tracks which estimation created the invoice
   - Maintains audit trail for financial reconciliation
