🔧 TROUBLESHOOTING GUIDE
═══════════════════════════════════════════════════════════════════════════════

Problem: Still seeing 404 errors
Solution:
  1. Kill all node processes: taskkill /f /im node.exe
  2. Wait 2 seconds
  3. Start server: start-dev.bat
  4. Wait for "All systems operational" message
  5. Clear browser cache: Ctrl+Shift+Delete
  6. Reload frontend: Ctrl+F5
  7. Check DevTools Network tab

Problem: Port 5000 already in use
Solution:
  taskkill /f /im node.exe
  OR identify process: netstat -ano | findstr :5000
  Then kill: taskkill /pid <PID> /f

Problem: Database connection error
Solution:
  1. Verify MySQL is running
  2. Check .env.development has correct credentials
  3. Test connection: node verify-all-apis.js (after server starts)
  4. Create database if missing: CREATE DATABASE deals_db;

Problem: Routes not loading
Solution:
  1. Check server.js has all route imports
  2. Verify route files exist in routes/ directory
  3. Restart server completely
  4. Check console output for errors
